# Cosmos Engine
# Decision Log

---

## 2026-07-07

### Decision

Documentation comes before implementation.

### Reason

The primary goal of Cosmos Engine is learning Physics and Cosmology through implementation.

Understanding must happen before coding.

### Impact

Every future feature requires:

1. Scientific understanding.
2. Persian documentation.
3. English documentation.
4. Implementation.

---

## 2026-07-07

### Decision

Knowledge Base created before continuing feature development.

### Reason

A significant portion of the project was implemented before the underlying scientific concepts were formally documented.

### Impact

Development is temporarily paused while scientific foundations are documented.

---

## 2026-07-10

### Decision

Project documentation is divided into stable and dynamic documents.

### Reason

Project philosophy changes slowly, while project status changes frequently.

Mixing both responsibilities in a single file would create maintenance problems.

### Impact

Project Charter contains stable project principles.

Project Status contains frequently changing project information.

Decision Log contains historical decisions.

---

## 2026-07-10

### Decision

Cosmos Engine will evolve beyond a physics simulator.

### Reason

The long-term vision is to explore not only known physical laws but also alternative physical models and hypothetical universes.

### Impact

Future architecture should remain flexible enough to support multiple physics models.
