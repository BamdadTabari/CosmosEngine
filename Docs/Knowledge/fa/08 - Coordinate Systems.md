# 08 — دستگاه‌های مختصات (Coordinate Systems)

---

# عنوان

دستگاه‌های مختصات (Coordinate Systems)

---

# هدف

درک مفهوم دستگاه مختصات، آشنایی با نقش آن در نمایش موقعیت اجسام و بردارها، و شناخت اهمیت آن در شبیه‌سازی‌های فیزیکی و Cosmos Engine.

---

# انگیزه

در فصل قبل یاد گرفتیم که بردارها دارای مقدار و جهت هستند.

اما اگر برداری مانند

```
(3, 5, -2)
```

را مشاهده کنیم، یک سؤال مهم مطرح می‌شود:

این سه عدد نسبت به چه چیزی اندازه‌گیری شده‌اند؟

بدون داشتن یک مرجع مشترک، هیچ برداری معنای مشخصی نخواهد داشت.

برای پاسخ به این سؤال، از دستگاه‌های مختصات استفاده می‌کنیم.

---

# سؤال

دستگاه مختصات چیست؟

چرا برای توصیف موقعیت اجسام به آن نیاز داریم؟

چگونه بردارها در یک دستگاه مختصات معنا پیدا می‌کنند؟

---

# شهود (Intuition)

فرض کنید دوستتان تماس می‌گیرد و می‌گوید:

"من در نقطه (10، 8) ایستاده‌ام."

اولین سؤال شما این خواهد بود:

"نسبت به کجا؟"

اگر هیچ مرجعی وجود نداشته باشد، این اعداد هیچ اطلاعات مفیدی منتقل نمی‌کنند.

اما اگر توافق کنیم که گوشه‌ی پایین سمت چپ یک نقشه نقطه‌ی (0,0) باشد، آنگاه همه می‌توانند مکان مورد نظر را پیدا کنند.

دستگاه مختصات دقیقاً همین مرجع مشترک را فراهم می‌کند.

---

# توضیح

دستگاه مختصات سیستمی است که به ما اجازه می‌دهد مکان نقاط، اجسام و بردارها را نسبت به یک مبدأ مشخص بیان کنیم.

هر دستگاه مختصات شامل موارد زیر است:

- یک مبدأ (Origin)
- یک یا چند محور (Axes)
- جهت مثبت هر محور
- واحد اندازه‌گیری

در فضای دوبعدی معمولاً از دو محور استفاده می‌شود:

- X
- Y

و در فضای سه‌بعدی:

- X
- Y
- Z

---

# دیدگاه علمی

در فیزیک، هیچ موقعیتی به صورت مطلق تعریف نمی‌شود.

همه‌ی موقعیت‌ها نسبت به یک دستگاه مختصات بیان می‌شوند.

به همین دلیل، وقتی مکان یک جسم را بیان می‌کنیم، در واقع مختصات آن را نسبت به یک مرجع مشخص اعلام می‌کنیم.

این موضوع یکی از پایه‌های اصلی مکانیک کلاسیک و مکانیک مداری است.

---

# ریاضیات

یک نقطه در فضای دوبعدی به صورت زیر نمایش داده می‌شود:

```
P = (x, y)
```

در فضای سه‌بعدی:

```
P = (x, y, z)
```

برای مثال:

```
Earth = (149,600,000 km, 0, 0)
```

یا

```
Spacecraft = (1200, 350, -80)
```

این اعداد تنها زمانی معنا دارند که دستگاه مختصات مشخص شده باشد.

---

# در Cosmos Engine

تمام اجسام موجود در Cosmos Engine دارای موقعیت هستند.

این موقعیت با استفاده از یک بردار سه‌بعدی ذخیره می‌شود.

برای مثال:

- Position
- Velocity
- Acceleration

همگی در یک دستگاه مختصات تعریف می‌شوند.

ساختار `Vector3D` در حقیقت مختصات یک نقطه یا یک بردار را در فضای سه‌بعدی نگهداری می‌کند.

در آینده، Cosmos Engine از چندین دستگاه مختصات مختلف نیز پشتیبانی خواهد کرد؛ مانند:

- World Coordinate System
- Local Coordinate System
- Orbital Coordinate System

---

# برداشت‌های نادرست رایج

### ۱. مختصات خودِ مکان هستند.

اشتباه.

مختصات تنها روشی برای توصیف مکان هستند، نه خود مکان.

---

### ۲. محورهای X ،Y و Z همیشه ثابت هستند.

اشتباه.

بسته به مسئله، می‌توان دستگاه‌های مختصات متفاوتی تعریف کرد.

---

### ۳. همه‌ی نرم‌افزارها از یک دستگاه مختصات استفاده می‌کنند.

اشتباه.

موتورهای مختلف ممکن است از قراردادهای متفاوتی برای جهت محورهای X، Y و Z استفاده کنند.

---

### ۴. مختصات بدون مرجع معنا دارند.

اشتباه.

هر مختصاتی باید نسبت به یک دستگاه مختصات تعریف شود.

---

# خلاصه

دستگاه مختصات چارچوبی برای توصیف مکان و جهت در فضا است.

بدون وجود یک دستگاه مختصات، بردارها و موقعیت اجسام معنای دقیقی نخواهند داشت.

تمام محاسبات فضایی در Cosmos Engine بر پایه‌ی دستگاه‌های مختصات انجام خواهند شد.
